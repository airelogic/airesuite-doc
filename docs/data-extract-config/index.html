    Data Extracts

    Add and configure the data extracts

    Jul 13, 2021

    This page explains how to add data extracts in AireFrame. These data extracts will contain the schema for a data source which will be used to create a data visualisation. See the guide on how to configure data sources here.

    What you can find on this page:

    • Required Permissions
    • Add Data Extract

    Required Permissions

    • In order to add a data extract you will need the following permissions:
      • List Data Extract
      • View Data Extract
      • Create Data Extract

    Add Data Extract

    1. Go to the data visualisation configuration screen: Configuration -> Data Visualisations.
    2. Click Add Data Extract, you will see a screen as follows
    3. Enter a title for the data extract
    4. Select the data source

    5. Select the data source that you wish to consume the data for your visualisation from.
      If you have chosen a non-internal data source, you will need to provide a relevant JSON extract definition for that data source (You will not be able to add the data extract unless you have provided the extract definition):

      • Forms4Health Example 
          {
  "key": "news2",
  "type": "building-block",
  "version": 1,
  "dataPoints": [
      {
          "controlId": "NEWSScore",
          "valueType": "displayValue"
      }
  ]
  }

        Form-level example

         {
      "type":"form",
      "key":"fluid-balance",
      "version":1,
      "dataPoints":[
          {
              "buildingBlock":{
                  "key":"fluid-intake",
                  "version":2
              },
              "controlId":"TotalFluidIntake"
          },
          {
              "buildingBlock":{
                  "key":"fluid-loss",
                  "version":1
              },
              "controlId":"TotalFluidLoss"
          },
          {
              "buildingBlock":{
                  "key":"net-fluid-balance",
                  "version":1
              },
              "controlId":"NetFluidIntake"
          }
      ]
  }

        Where:

        • valueType (optional, defaults to value) is value (underlying value) or displayValue (display text value)
      • AireFlow Example 
          {
  "status": "NEW"
  }

        Where:

        • status (optional, if omittied returns all tasks) is the Tasks’ status. Available options are NEW, IN_PROGRESS and COMPLETED
      • FHIR R4 Example 
          {
  "resourceType": "Observation",
  "searchParameters": [
      "code-value-concept=http://snomed.info/sct|364075005"
  ],
  "dimensions": [
      {
          "key": "date",
          "fhirPath": "Observation.effective"
      },
      {
          "key": "heartRate",
          "fhirPath": "Observation.value.value"
      }
  ]
  }

        Where:

        • The resource type is what you want to query from the FHIR server - it must be searchable by patient identifier
        • Search parameters are used to filter the resource type, in the example this is for the snomed code for heart rate observations. You can use as many as you like and use normal FHIR conventions (and/or) etc.
        • The dimensions are used to populate the widget with the fetched data, the keys are the data points that you want to show, they are resolved using FHIRPath, so any FHIRPath query is valid here. You can use FHIRPath to combine multiple expressions in case one or more do not resolve in the fetched data. The value returned for each data point will be the first FHIRPath expression that finds a value.

    6. Click create
