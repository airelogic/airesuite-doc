    Getting Started

    Jun 22, 2021

    The first time that you log into a new tenant on aireframe, you will be presented with this screen:

    This is not an error, it just means that the tenant needs to be configured first. Click the cog icon in the bottom left to start configuring the tenant.

    Organisational Structure

    AireFrame allows an organisation to flexibly define how their structure should be represented. This structured defines where the subjects can be assigned, so you must set this up before you can add subjects.

    See the guide on how to setup an organisation structure here.

    Groups

    Groups define the mapping between users and the organisation structure, i.e. users can only access parts of the organisation that they have been assigned to via groups. Once you have assinged your user to a group, the initial access denied message will dissapear.

    See the guide on how to setup groups here.

    Data Source

    The next step is to setup a data source from where subjects will be served. The default data source for this is the ‘Internal’ data source, where the subjects are stored in the cloud alongside AireFrame’s other data, however it is possible to use subjects served from a FHIR store.

    See the guide on how to configure data sources here.

    Subject Configuration

    The final step is to update the subject configuration. This will allow you to choose which data source to serve subjects from, set the subject name and decide what fields to associate with a subject.

    See the guide on how to update the subject provider here.
    See the guide on how to update the subject configuration here.
