    Setting up and modifying organisational structure strcuture

    Mar 11, 2021

    This page explains how manage organisation structure in AireFrame. The powerful recursive system allows you to configure the structure in aireframe to the needs of your organisation

    What you can find on this page:

    • Required Permissions
    • Structure Types
    • Structural Entities

    Required Permissions

    • In order to add a structural entity/type you will need the following permissions:
      • List Structural Entities
      • Create Structural Entity
    • In order to edit a structural entity/type you will need the following permissions:
      • List Structural Entities
      • View Structural Entity
      • Edit Structural Entity

    Structure Types

    Structure types are the possible ‘types’ that you would like in your organisation structure. For example, this could be ‘Hospital’, ‘Ward’ or ‘Clinic’.

    Structure types have config that limit what they can be used for - such as whether or not you can assign a subject to them (subject locations). Every structural entity must be of one particular type.

    1. Go to the structure configuration screen: Configuration -> Structure.
    2. Click Add Structure Type, you will see a screen as follows
    3. Enter a name for the structure type
    4. Choose the structure type configuration
    5. Click save

    Structure types can be edited by clicking the arrow at the right hand side of the structure type listing. Please note, that configuration changes do not affect the current data. i.e. if a subject is assigned to a structural entity of a structure type and that type is changed to not be subject assignable, the subject is not removed from that structural entity

    Structural Entities

    Structural entities make up the actual organisational structure that you see in AireFrame. It is a recursive structure so it is possible to have a deep nested structure of many different structure types, with user access control provided by the groups feature.

    1. Go to the structure configuration screen: Configuration -> Structure.
    2. This shows a structure tree that starts at the top level and can be expanded to see all the children.
    How to add Structural Entities
    1. Go to the structure configuration screen: Configuration -> Structure.
    2. Click Add icon, you will see a screen as follows
    3. Enter a display name, unique key and structure type for the new structure entity. If it has a parent, then it can be added under it’s parent entity using the Structural Entity Search.
    4. Click save.

    Note:

    • Each structural entity that is created must have a unique key - this is so that external integrations can reference structural entities in a tenant.
    • Once created, a structural entity cannot be moved to a different parent due to access controls.
    • Structural entities cannot currently be deleted
    • If no parent entity is selected in the Structural Entity Search the default parent will be the root tenant.
