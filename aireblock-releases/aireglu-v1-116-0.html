AireGlu Release Notes ​

Version: 1.116.0
Release Date: 21st April 2026

Summary of Changes ​

✨ New Features ​

  • Automatic AireForms Schema Import for Endpoints (GLU-2873)
  • Connect AireGlu Endpoints to AI Assistants via MCP (GLU-2904)
  • Tenant-Wide Client Credential Connections for HTTP Request Authentication (GLU-2998)

⚡ Improvements ​

  • Unsaved Changes Warning in Endpoint Editor (GLU-2813)
  • Improved Auto-Generated Schema Validation (GLU-2945)
  • Expanded Retry Support for Endpoint-to-Endpoint Calls (GLU-2967)
  • Usage Data Table Refresh Button (GLU-2915)
  • Retry Activity in Metrics (GLU-3007)
  • Narrowed Retry Conditions to Specific Exceptions (GLU-3024)

🐞 Bug Fixes ​

  • Job Re-activation Runs Immediately (GLU-2955)
  • JSON Editor Container Overflow on JWT Headers (GLU-2900)
  • Config Assistant Broken After BFF Migration (GLU-2996)
  • Improved Error Message for Empty XML Schema Save Failure (GLU-3036)
  • Search Close Opens All Groups (GLU-3016)

Detailed Changes ​

✨ New Features ​

Automatic AireForms Schema Import for Endpoints (GLU-2873) ​

Description

The endpoint editor can now automatically pull in AireForms schemas as the input schema for an endpoint, removing the need to define or copy schemas manually.

What this means for you

When configuring an endpoint that receives data from an AireForm, you can now import the form's schema directly into the editor with a single action, saving time and reducing the risk of mismatches between your form and endpoint configuration.

Connect AireGlu Endpoints to AI Assistants via MCP (GLU-2904) ​

Description

AireGlu endpoints can now be connected to AI assistants using the Model Context Protocol (MCP), enabling AI tools to discover and invoke endpoints as part of an agentic workflow.

What this means for you

You can now expose your AireGlu endpoints to AI assistants, allowing them to be called as tools within AI-driven processes — opening up new possibilities for automation and intelligent workflows across your organisation.

Tenant-Wide Client Credential Connections for HTTP Request Authentication (GLU-2998) ​

Description

Tenant-wide client credential connections can now be configured and used to authenticate HTTP requests within endpoints, providing a centralised and reusable approach to securing outbound API calls.

What this means for you

You can now set up a shared client credential connection once at the tenant level and apply it to HTTP requests across multiple endpoints, reducing repetitive configuration and making it easier to manage authentication credentials in one place.

⚡ Improvements ​

Unsaved Changes Warning in Endpoint Editor (GLU-2813) ​

Description

The endpoint editor now displays a warning when navigating away with unsaved changes, preventing accidental loss of work.

What this means for you

You'll be prompted before leaving the editor if you have unsaved changes, giving you the chance to save or discard before moving on.

Improved Auto-Generated Schema Validation (GLU-2945) ​

Description

The minimum items requirement has been removed from automatically generated JSON schemas in the endpoint editor, preventing schemas from incorrectly failing their own validation.

What this means for you

Auto-generated schemas will now validate correctly out of the box, reducing configuration errors when setting up endpoints that use automatically derived input schemas.

Expanded Retry Support for Endpoint-to-Endpoint Calls (GLU-2967) ​

Description

Retry logic has been extended to cover calls made from one AireGlu endpoint to another, improving resilience when transient failures occur between endpoints.

What this means for you

AireGlu will now automatically retry failed endpoint-to-endpoint calls, reducing the impact of temporary disruptions and making your integrations more reliable.

Usage Data Table Refresh Button (GLU-2915) ​

Description

A refresh button has been added to the usage data table in the metrics view, allowing the latest data to be loaded without navigating away from the page.

What this means for you

You can now reload your usage data on demand with a single click, giving you up-to-date information without having to leave and return to the metrics view.

Retry Activity in Metrics (GLU-3007) ​

Description

Retry attempts are now recorded and displayed within the metrics view, providing visibility into how often tasks are retried and their outcomes.

What this means for you

You can now see retry activity alongside your other metrics, helping you identify unreliable integrations and understand how often automatic retries are occurring.

Narrowed Retry Conditions to Specific Exceptions (GLU-3024) ​

Description

The retry mechanism has been updated to trigger only on specific, well-defined HTTP exceptions rather than broadly on any failure, reducing unnecessary retries and improving predictability.

What this means for you

Retries will now only occur when genuinely appropriate, making the behaviour of your integrations more consistent and easier to reason about.

🐞 Bug Fixes ​

Job Re-activation Runs Immediately (GLU-2955) ​

Description

Fixed an issue where re-activating a scheduled job would cause it to run immediately if it had run previously, rather than waiting for its next scheduled time.

JSON Editor Container Overflow on JWT Headers (GLU-2900) ​

Description

Fixed an issue where the JSON editor would overflow its container when JWT headers were displayed, causing layout problems in the endpoint editor.

Config Assistant Broken After BFF Migration (GLU-2996) ​

Description

Fixed an issue where the endpoint editor config assistant stopped functioning following the BFF (Backend for Frontend) migration, restoring full assistant functionality.

Improved Error Message for Empty XML Schema Save Failure (GLU-3036) ​

Description

Improved the error message shown when an endpoint with an empty or invalid XML schema fails to save, making it clearer to the user that the save was unsuccessful.

Search Close Opens All Groups (GLU-3016) ​

Description

Fixed a bug where closing a search in the endpoints list would unexpectedly expand all collapsed groups, disrupting the user's view.
